# Begin this activity by...
> Having a scrum meeting!
>
> Your team should create a project board and then create cards and issues for each task. This is repetitive and is meant to be used as practice. Start your first sprint by prioritizing the first easy tasks(letters as an example) people can access; once everyone is done with their tasks, close the card & issues and then move to the next issue.
>
> Practice branching in your repository by creating a new branch for every task you work on. For example, if you personally get tasked with the letter C - F, then create a branch called "letterC-F". Once you finish those letters and are done with the branch, make sure to create a new branch for your next letters. For example, if you are next tasked with letterG-I, your branch would be "letterG-I".

## Useful commands for creating and moving to branches...
    - `git branch` (shows current local branches)
    - `git branch branchName` (creates a new branch)
    - `git checkout branchName` (moves you to the branch listed)
    - `git checkout -b branchName` (creates a new branch and moves you to it)

## How to push a branch... 
    - `git status`
    - `git add .`
    - `git commit -m 'messages'`
    - `git status`
    - `git pull origin main`
    - `git push origin branchName`

# Comiencen esta actividad...
> ¡Teniendo una reunión de Scrum!
>
> Su equipo debe crear un tablero de proyecto y luego crear tarjetas e incidencias (issues) para cada tarea. Esto es repetitivo y está destinado a usarse como práctica. Comiencen su primer sprint priorizando las primeras tareas fáciles (letras como ejemplo) a las que la gente pueda acceder; una vez que todos terminen con sus tareas, cierren la tarjeta y las incidencias y luego pasen a la siguiente incidencia.
>
> Practiquen el uso de ramas (branching) en su repositorio creando una nueva rama para cada tarea en la que trabajen. Por ejemplo, si personalmente te asignan las letras C - F, entonces crea una rama llamada "letraC-F". Una vez que termines esas letras y hayas terminado con la rama, asegúrate de crear una nueva rama para tus siguientes letras. Por ejemplo, si luego te asignan las letras G-I, tu rama sería "letraG-I".

## Comandos útiles para crear y moverse a ramas...
    - `git branch` (muestra las ramas locales actuales)
    - `git branch nombreRama` (crea una nueva rama)
    - `git checkout nombreRama` (te mueve a la rama listada)
    - `git checkout -b nombreRama` (crea una nueva rama y te mueve a ella)

## Cómo subir una rama (push)...
    - `git status`
    - `git add .`
    - `git commit -m 'mensajes'`
    - `git status`
    - `git pull origin main`
    - `git push origin nombreRama`
